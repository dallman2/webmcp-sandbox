Audit: Baseline Quality

Epic: v0.0-baseline Date: 2026-06-26 Sprint 0 baseline audit

Executive Summary

This audit characterizes the webmcp-sandbox scaffold as it exists at the start of Sprint 0. The Docker image (webmcp-sandbox:latest, 836MB) boots and serves a stub SSE endpoint on :3000, and VNC is reachable on :5900. However, the system cannot function end-to-end for three independent reasons: (1) the Rust server lacks the rmcp crate, schemars, tool definitions, and a :8765 WebSocket listener — the extension's relay has nothing to connect to; (2) Chromium crashes on boot because the Ubuntu 22.04 chromium-browser package is a snap wrapper shim with no snapd available in the container; (3) the codebase has zero tests and multiple AGENTS.md mandate violations (no structured logging fields, no Zod in the extension, no DESIGN.md). On the positive side: the extension's background.ts/content.ts is the most complete component, correctly implementing the Golden Rule dispatch pattern with @mcp-b packages; the Rust server's tracing-to-stderr discipline is correct; and both halves build cleanly (WXT and cargo build --release).

Strengths

  • Extension Golden Rule compliance: background.ts and content.ts exclusively use document.modelContext.executeTool() and .getTools() — zero eval() or manual DOM traversal. Grep across src/ confirms zero matches for eval\( or page\.evaluate.
  • tracing → stderr is correct: main.rs:45 uses .with_writer(std::io::stderr). Zero println! calls in the Rust source. The stream-corruption guard is already in place.
  • Docker build is fully cacheable: all 3 stages use layer caching correctly (Cargo.toml/package.json copied first, then source). Rebuilds are fast.
  • Wire protocol is already defined and typed: background.ts:16–70 defines the JSON envelope format the server must implement — call_tool, discover_tools, tool_result, tool_error, discover_result — with id-based correlation. The server side is the only missing half.
  • start.sh is 29 lines: meets the ≤30-line mandate from AGENTS.md:106.
  • WXT build succeeds: outputs .output/chrome-mv3/ with manifest, background.js, and content-scripts in 465ms.

Weaknesses (Critical Risks)

WeaknessRisk LevelDescription
Chromium snap wrapper crash Critical Ubuntu 22.04 chromium-browser package installs a shell script that tries to exec /snap/bin/chromium. Snap is not available in the Docker container (no systemd, no snapd). Chromium exits immediately with the "requires the chromium snap" error. The browser never starts; CDP :9222 returns connection refused; the extension is never loaded. This blocks all end-to-end testing — independent of the relay gap.
No :8765 WebSocket listener Critical mcp-server/src/main.rs has no WS server. The extension's background.ts:2 connects to ws://127.0.0.1:8765 — confirmed closed (connection refused on port probe). start.sh:11 log message says "starting MCP translation server (SSE :3000, WS :8765)" but only :3000 is bound. The relay is the single largest server-side gap.
rmcp + schemars absent Critical Neither rmcp nor schemars appear in mcp-server/Cargo.toml. The SSE endpoint is a raw axum::sse::Sse::new(stream::iter(...)) emitting one hard-coded notifications/initialized event (not real MCP). AGENTS.md:92 mandates rmcp = { features = ["server","transport-io","transport-sse-server"] } — entirely missing.
src/tools.rs does not exist High mcp-server/README.md:15 claims src/tools.rs contains #[rmcp::tool] stubs for discover, get_page_state, relay. The file does not exist. The only source file under mcp-server/src/ is main.rs. No #[rmcp::tool] macros exist anywhere in the repo.
No structured logging fields High AGENTS.md:59 mandates correlation_id, tool_name, and ws_session_id in all log events. main.rs only uses flat info! calls — no span fields are attached. This is testable but unimplemented.
Zero test coverage Medium cargo test compiles but runs 0 tests (no #[test] in any .rs file). Extension has no test script in package.json and no vitest dependency. AGENTS.md:101 defers extension unit tests to Sprint 1 — acknowledged.
Zod validation missing Medium AGENTS.md:65 mandates "Zod validation for extension-side tool arguments before forwarding to the page." No zod import, dependency, or validation logic exists in the extension source. Grep returned zero matches.
DESIGN.md not created Low AGENTS.md:160 lists DESIGN.md as a reference file "needs creation in Sprint 0." File does not exist.
--transport stdio parsed but unused Low main.rs:13 declares a --transport clap arg (default "sse") but main.rs:48 never branches on it — only the SSE path exists. The transport-io rmcp feature (mandated) would drive this.
tsc type errors in extension Low npx tsc --noEmit reports 6 errors: defineBackground/defineContentScript not found (WXT globals missing type declarations) and document.modelContext not on Document type (@mcp-b/webmcp-types types not resolved). WXT build succeeds regardless because WXT handles its own type resolution. Cosmetic for now but should be addressed in the tsconfig.
No Cargo.lock committed Low The repo has no Cargo.lock and no package-lock.json. gitignore should be reviewed; for a binary crate, Cargo.lock should typically be committed.

Recommendations

Immediate (Sprint 1)

  • Fix Chromium runtime: replace the Ubuntu 22.04 snap wrapper with a direct Chromium binary (e.g. from Google's PPA, a downloaded .deb, or switch the runtime base to Debian bookworm which has a real chromium package). This unblocks all browser-dependent testing.
  • Add rmcp + schemars: wire the mandated rmcp SSE transport, replace the stub, and create src/tools.rs with #[rmcp::tool] definitions using schemars-derived JSON schemas.
  • Add :8765 WS listener: reuse axum's ws feature (no new crate) to bind the relay. Forward call_tool/discover_tools envelopes matching background.ts's wire protocol exactly.
  • Add structured logging fields: correlation_id (= JSON-RPC request id), tool_name, ws_session_id on all tool-call log events.
  • Fix mcp-server/README.md drift: update to reflect actual code once src/tools.rs exists.

Strategic (Future Sprints)

  • Add test coverage: introduce vitest for the extension and #[test] for the Rust server. Minimum bar: a single end-to-end smoke test proving the discover→call→respond loop.
  • Implement --transport stdio: once rmcp's transport-io feature is wired, branch on the already-parsed --transport arg.
  • Add Zod validation: wrap tool arguments in background.ts with Zod schemas before forwarding to document.modelContext.executeTool().
  • Fix WXT type declarations: configure tsconfig to resolve defineBackground/defineContentScript globals and document.modelContext types so tsc --noEmit passes clean.
  • Create DESIGN.md: document design tokens — referenced by AGENTS.md but deferred.
