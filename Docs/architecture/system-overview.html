Project Purpose

WebMCP Sandbox is a containerized, ephemeral browser sandbox for AI agents using the Web Model Context Protocol (WebMCP). The agent discovers and calls web-page tools through strictly typed JSON payloads — never raw page.evaluate JavaScript. A Rust MCP server translates between the AI agent (connecting via SSE on :3000) and a headless Chromium browser (communicating through an MV3 Chrome extension relayed over WebSocket on :8765). The target web page exposes tools through document.modelContext, injected by the extension's polyfill.

Data Flow (discover → call → respond)

  1. Agent connects via SSE to http://localhost:3000/sse — the rmcp SSE transport (stubbed today, real in Sprint 1).
  2. Agent discovers tools: the server forwards a {type:"discover_tools", id} JSON envelope over the :8765 WebSocket to the browser extension's background service worker.
  3. Extension relays to page: background.ts uses chrome.scripting.executeScript to call document.modelContext.getTools() on the target tab. The polyfill (@mcp-b/webmcp-polyfill, injected by content.ts) provides the implementation if the page doesn't natively expose document.modelContext.
  4. Tools returned: the extension sends {type:"discover_result", tools:[...], id} back over WS; the server republishes to the SSE client.
  5. Agent calls a tool: {type:"call_tool", id, toolName, args} over WS → background.ts calls document.modelContext.executeTool(toolName, JSON.stringify(args)) on the page.
  6. Native response: {type:"tool_result", id, result} (or tool_error) flows back over WS; server republishes to SSE.

Wire protocol (server ↔ extension, defined by background.ts:16–70): JSON messages with {type, id, ...} — types are call_tool, discover_tools, tool_result, tool_error, discover_result. The id field correlates request-response pairs across the relay.

The Golden Rule & Mandate Compliance (Sprint 0 baseline)

The agentic harness must never send JavaScript to the browser. It must only send strictly typed JSON payloads through the MCP server. All interactions with the page go through document.modelContext.executeTool() and document.modelContext.getTools() — never eval() or manual DOM traversal.
