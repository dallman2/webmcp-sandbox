Sprint 0: Bootstrap & Verification

Phase: 0 Sprint: 0 Depends on: —

Objective

Characterize the current scaffold state of webmcp-sandbox against the webmcp-sandbox:latest Docker image, produce the missing architecture and audit documentation, and define a factual baseline (lint, typecheck/build, test coverage) so that Sprint 1 can implement the end-to-end bridge with a clear acceptance surface.

Rationale & Context

The repo is at scaffold stage: an 836MB Docker image exists and boots, the extension bridge (background.ts/content.ts) is the most complete component, but the Rust server is a stub — no rmcp dependency, no #[rmcp::tool] definitions, no WebSocket listener on :8765, and a hard-coded one-shot SSE event. The in-repo documentation (README claims of src/tools.rs) is drifted from the actual code. This sprint quantifies that state without writing new feature code, and feeds its findings into Sprint 1.

Scope

In-Scope

  • Run webmcp-sandbox:latest and probe ports 3000 (SSE), 5900 (VNC), 9222 (CDP), and 8765 (WS, expected absent)
  • Confirm the extension ECONNREFUSED retry loop is observably the reason the bridge cannot function today
  • Audit README/AGENTS vs. code drift (e.g. mcp-server/README.md referencing a non-existent src/tools.rs)
  • Author Docs/architecture/system-overview.html
  • Run Rust lint/build (cargo fmt --check, clippy -D warnings, cargo build --release) in builder Docker stages — no host toolchain required
  • Run extension lint/build (npx tsc --noEmit, npm run build) in builder Docker stages
  • Record test-suite baseline (currently zero tests / no runner configured)
  • Author Docs/audits/baseline-quality.html capturing all findings
  • Decide the next epic from the audit — input to Sprint 1

Out-of-Scope

  • Adding rmcp/schemars, WS listener, or tool definitions — that is Sprint 1
  • Fixing lint/typecheck issues surfaced by the audit — Sprint 1 or later
  • Stand-up of a host cargo/node toolchain — all audits run inside the Docker builder stages

Execution Plan

1. Scaffold Verification

  1. Task 1.1: Run current image, confirm SSE/VNC/CDP come up

    Run docker run -d --name webmcp-verify -p 3000:3000 -p 5900:5900 webmcp-sandbox:latest, then from the host probe: curl -N http://localhost:3000/sse (expect the one-shot notifications/initialized stub event + 15s keep-alive per mcp-server/src/main.rs), nc -z localhost 5900 (VNC), and docker exec webmcp-verify curl -s http://127.0.0.1:9222/json/version (CDP, internal-only by design). Record exact observed payloads/timing into the audit doc.

  2. Task 1.2: Confirm the :8765 WS gap and extension ECONNREFUSED loop

    From inside the container, docker exec webmcp-verify sh -c 'curl -sS http://127.0.0.1:8765' must fail (port not bound). Tail docker logs -f webmcp-verify and confirm the extension's auto-reconnect retry storm from background.ts. This pins the documented Sprint-1 cliff: the relay is the single biggest functional gap.

  3. Task 1.3: README/AGENTS vs. code drift audit

    Inventory statements in mcp-server/README.md and AGENTS.md that describe files/features that do not exist (notably mcp-server/README.md:15 claims src/tools.rs with #[rmcp::tool] stubs — the file is absent). Capture the list verbatim into Docs/audits/baseline-quality.html and note that check-doc-drift.mjs will not catch it (prose-only).

2. Architecture Documentation

  1. Task 2.1: Write system overview architecture doc

    Create Docs/architecture/system-overview.html from Docs/templates/reference.html describing: project purpose (containerized ephemeral browser sandbox for WebMCP agents), the 3-component topology (Rust MCP server, MV3 extension, Docker runtime), data flow (agent→SSE:3000→rmcp→WS:8765→extension→document.modelContext→page), port table, the Golden Rule (no JS injection), and the known gap that Sprint 1 closes. Reflect actual scaffold state, not aspiration.

3. Codebase Audits (Docker builder stages)

  1. Task 3.1: Rust lint audit — cargo fmt + clippy

    Build the rust-builder stage in isolation and run lint checks within it, avoiding host toolchain setup: docker build --target rust-builder -t ws-audit-rust -f docker/Dockerfile . then docker run --rm ws-audit-rust sh -c 'cargo fmt --check && cargo clippy --all-targets -- -D warnings'. Record pass/fail and any specific findings (unused args like the parsed-but-unbranched --transport flag).

  2. Task 3.2: Typecheck/build audit — Rust + extension

    For Rust: docker run --rm ws-audit-rust sh -c 'cargo build --release'. For the extension: docker build --target ext-builder -t ws-audit-ext -f docker/Dockerfile . then docker run --rm ws-audit-ext sh -c 'npx tsc --noEmit && npm run build'. Record whether each builds and any non-zero exit artifacts. Note the absence of a committed Cargo.lock and extension lockfile.

  3. Task 3.3: Test-suite baseline

    Establish the factual baseline: zero #[test] in mcp-server/src/; no vitest dependency and no test script in webmcp-extension/package.json; AGENTS.md defers extension unit tests to Sprint 1. The audit finding for this task is "0 tests, 0 runners configured" — a documented baseline, not a failure.

  4. Task 3.4: Author baseline-quality audit doc

    Write Docs/audits/baseline-quality.html from Docs/templates/audit.html consolidating the findings of tasks 1.1–1.3 and 3.1–3.3 into a structured audit: what boots, what is stubbed, lint/typecheck/build outcomes, test baseline, drift list, and the Sprint-1 priorities that follow.

4. Planning

  1. Task 4.1: Identify next epic — draft Sprint 1 scope

    From the audit findings, confirm the next chunk of work is implementing the end-to-end bridge: rmcp+schemars deps, a src/tools.rs with #[rmcp::tool] definitions, replacing the stub SSE handler, adding the :8765 WS relay (reusing axum's ws feature, no new crate), full structured logging (correlation_id/tool_name/ws_session_id), an E2E fixture, and a stdout-clean smoke test. Sprint 1's plan is authored from this task; task checkboxes here flip as Sprint 1 work lands.
