Epic Overview: v0.1 — Exercise Hardening

Epic Vision

Harden the v0.0 baseline by resolving the SSE-transport doc-vs-code divergence (replace the hand-rolled legacy SSE handler with canonical rmcp::StreamableHttpService), fix container runtime failures (missing fixtures, missing networking), prove the sandbox works end-to-end against a real WebMCP-enabled site at localhost:8180, and deliver single-command portability (make run from a fresh clone).

This epic makes the project real: a real MCP client can connect, a real target site can be exercised, and a new user can git clone && make run to get a working sandbox.

Acceptance Criteria

  • MCP inspector connects to http://localhost:3000/mcp via Streamable HTTP transport
  • tools/call execute_tool returns a real result (not null) from the fixture page
  • Docker container boots all services, fixture page loads, tool call round-trips end-to-end
  • Container can reach a host-side localhost:8180 on both macOS and Linux
  • make smoke-fixture passes all assertions against the fixture page
  • make smoke-host passes assertions against user's localhost:8180 site
  • git clone && make run produces a working sandbox (single command)
  • validate-docs.mjs passes with zero violations
  • check-doc-drift.mjs passes with zero drifts
